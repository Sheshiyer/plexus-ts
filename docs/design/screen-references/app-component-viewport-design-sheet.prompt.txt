Generate a single high-fidelity Figma-style component and viewport design sheet for the Plexus app at 1536x1024.

This is not a full app screenshot. It is a reproducible design-system sheet for the non-Settings pages. Settings already has its own component sheet, so use it only as a consistency reference. The goal is to clean up a functional but messy UI by defining shared viewport archetypes and reusable components.

Use the attached current app screenshots as factual references for the existing pages. Use the attached Settings component sheet as the style and documentation baseline. Use the onboarding/field imagery only as a subtle visual philosophy reference: Plexus is a work coordination field, not a generic SaaS dashboard.

Product goal:
- Keep Plexus functional and operational.
- Make the app feel like one instrument with several modes.
- Preserve the current shell, HUD, sidebar, hard edges, mono telemetry, chartreuse/mint palette, and FORMA operator-console aesthetic.
- Reduce the messy feeling caused by one-off page layouts, random rows, stat-card clusters, and inconsistent empty/error states.

Brand and style rules:
- Backgrounds: #001417, #00272B, #012F34.
- Accent: #E0FF4F. Mint text: #D6FFF6.
- Violet only as a faint field trace. Rose only for actual error, inaccessible proof, denied permission, or destructive confirmation.
- Geist and Geist Mono.
- Hard square edges, 1px hairlines, bracket marks, crosshair corners.
- Inset glow only. No outer neon. No glassy SaaS blur cards. No rounded bento cards.
- Dense but orderly. Technical, calm, precise, alive.
