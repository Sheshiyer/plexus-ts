Redesign direction:
- Stop making the whole page a spreadsheet. Replace repeated generic tables with purpose-built operator modules.
- Use a left-side vertical section index inside the main Settings content: Identity, Profile, Proof, Setup, Runtime, Rhythm, Release.
- Use a hero identity module at the top: profile card on the left, verified session summary on the right, and compact status chips for Access, Worker, Bridge, Permissions, and GitHub proof.
- Use a reusable datum rail style: label, primary value, optional secondary value, status chip, and overflow-safe treatment. Long emails, URLs, token expiry timestamps, identity IDs, and endpoints must use middle truncation or two-line wrapping, never ugly clipping.
- Use tables only for comparable metrics. For bridge and session proof, use stacked key/value strips. For setup state, use compact readiness tiles. For permissions, use three grant pills. For sound/rhythm, use control rows with meters and toggles.
- Avoid card-inside-card feeling. Use full-width bands, hairline separators, and crosshair panel corners.
- The page should fit the visible app viewport better than the current long scroll. Show the top 70% of the redesigned page with enough below-the-fold hint to imply more sections.

FORMA visual contract:
- Gun-metal surfaces: #001417, #00272B, #012F34. Never pure black.
- Single chartreuse accent: #E0FF4F. Pale mint text: #D6FFF6 with opacity hierarchy. Violet only as subtle trace detail. Rose only for errors.
- Geist for body and headings. Geist Mono for labels, numerics, telemetry, status codes, and values.
- All-caps mono section labels with tracked letter spacing.
- 1px mint hairlines, inset chartreuse glow only. No outer neon glow, no drop shadows, no gradient text.
- Subtle scanline atmosphere. Crosshair corners on raised panels.
- Compact operational density but polished spacing. Text must never overlap, clip awkwardly, or escape containers.
- No emojis, no decorative stock imagery, no fake logos, no watermarks.
