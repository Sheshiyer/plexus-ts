# Plexus v0.4.0 No-Placeholder Quality Gate

Date: 2026-06-18
Scope: user-visible Plexus renderer surfaces, Fabric/Paperclip onboarding helpers, and release-blocking co-working controls.

## Gate Definition

A user-visible placeholder is any label, value, action, status, or generated file that looks like real product state but is not backed by current app data, a configured integration, a real local file, or an honest empty/error state.

Allowed states:
- Loading, when it resolves or reports failure.
- Empty states that name the missing data.
- Setup states that explain the missing configuration.
- Input fields with labels and accessibility names, without example text masquerading as saved data.

Blocked states:
- Static agent statuses generated by setup scripts.
- Blank profile fields seeded as if they were meaningful member data.
- Release UI wording that says demo or emulation.
- Generic `unknown`, `default`, `Untitled`, `probe pending`, or `not scanned` copy shown as product state.
- Co-working lounge entry without visible mic, camera, screen share, and leave controls.
- Fake GitHub repo, fake GitHub activity, fake biorhythm, fake breakwork audio, or placeholder appraisal data shown as if it were real proof.
