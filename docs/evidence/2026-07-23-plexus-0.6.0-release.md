## GitHub linking boundary

The main-process URL gate now accepts the documented optional GitHub OAuth
parameters through an explicit allowlist and validates values, duplicate keys,
PKCE pairing, exact callback, state, and GitHub host/path. Renderer and preload
surfaces do not receive repository authority or raw authorization URLs.

This proves local URL construction and rejection behavior only. The Workspace
Worker must still complete installation synchronization and repository
verification against a real private repository before the live link is called
end-to-end verified.

## Project media decision

Project mic, camera, and screen remain a deferred shell. The release does not
claim SFU transport because transport readiness, Cloudflare Realtime Worker
credentials, and two-participant E2E proof are not present in this candidate.
The lounge transport is a separate capability and is not evidence for project
media.
